### **Arrays**

#### **Advantages**:
1. **Random Access**: Arrays allow direct access to elements using their indices, making them faster for accessing elements (`O(1)`).
2. **Memory Locality**: Arrays are stored in contiguous memory, improving cache performance and access speed.
3. **Static Size (Predictable Memory)**: The size of an array is fixed, which makes memory management simpler and reduces overhead.

#### **Disadvantages**:
1. **Fixed Size**: The size of an array must be declared at the time of creation, leading to possible memory wastage or the need for resizing (which is expensive).
2. **Insertion/Deletion Cost**: Adding or removing elements, especially in the middle, is costly (`O(n)`) as it requires shifting elements.
3. **Inefficient Use for Dynamic Data**: Arrays are not well-suited for scenarios where frequent resizing or dynamic size changes are required.

---

### **Linked Lists**

#### **Advantages**:
1. **Dynamic Size**: Linked lists can grow or shrink in size dynamically, eliminating the need for resizing.
2. **Efficient Insertion/Deletion**: Adding or removing elements at the beginning or middle is efficient (`O(1)` for head operations).
3. **No Wastage of Memory**: Memory is allocated as needed, avoiding the potential for unused allocated space.

#### **Disadvantages**:
1. **Sequential Access**: Accessing elements is slow (`O(n)`) as traversal is required from the head to the desired node.
2. **Higher Memory Overhead**: Each node requires additional memory for a pointer/reference to the next node.
3. **Poor Cache Performance**: Nodes are scattered in memory, leading to inefficient caching and slower access times compared to arrays.

---

### **When to Use**:
- Use **arrays** when:
  - You need fast, random access to elements.
  - The size of the data structure is fixed or changes infrequently.
  - Memory locality and performance are critical.

- Use **linked lists** when:
  - The size of the data structure changes frequently.
  - You need to frequently insert/delete elements at arbitrary positions.
  - Memory efficiency (without pre-allocation) is important.

Each structure has its specific use cases, so the choice depends on the requirements of your application.

// Define the structure of a node
struct Node {
    int data;
    struct Node* next;
};

// Function to create a new node
struct Node* createNode(int data) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = data;
    newNode->next = NULL;
    return newNode;
}

// Function to reverse the linked list recursively
struct Node* reverseRecursive(struct Node* head) {
    // Base case
    if (head == NULL || head->next == NULL) {
        return head;
    }

    // Recursive case: reverse the rest of the list
    struct Node* newHead = reverseRecursive(head->next);

    // Adjust pointers
    head->next->next = head;
    head->next = NULL;

    return newHead;
}

// Function to print the linked list
void printList(struct Node* head) {
    struct Node* temp = head;
    while (temp != NULL) {
        printf("%d -> ", temp->data);
        temp = temp->next;
    }
    printf("NULL\n");
}

// Function to append a node at the end
void appendNode(struct Node** headRef, int data) {
    struct Node* newNode = createNode(data);
    if (*headRef == NULL) {
        *headRef = newNode;
        return;
    }
    struct Node* temp = *headRef;
    while (temp->next != NULL) {
        temp = temp->next;
    }
    temp->next = newNode;
}

// Main function
int main() {
    struct Node* head = NULL;

    // Create linked list: 1 -> 2 -> 3 -> 4 -> NULL
    appendNode(&head, 1);
    appendNode(&head, 2);
    appendNode(&head, 3);
    appendNode(&head, 4);

    printf("Original List:\n");
    printList(head);

    // Reverse the linked list
    head = reverseRecursive(head);

    printf("Reversed List:\n");
    printList(head);

    return 0;
}
